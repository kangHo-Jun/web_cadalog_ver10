# ERP-쇼핑몰 상품 자동 업로드 시스템 통합 개발 계획 (Modular v2)

## 📋 프로젝트 개요
이카운트 ERP 품목 정보를 기반으로 마진율을 적용하여 카페24 쇼핑몰에 상품을 자동으로 업데이트하는 전문적인 모듈형 시스템을 구축합니다.

---

## 🏗️ 왜 모듈화(Modularization)가 유리한가?
단일 스크립트보다 모듈형 개발이 유리한 이유는 다음과 같습니다:

1. **유지보수 용이성 (Maintainability)**: 카페24 API 명세나 이카운트 로그인 방식이 변경되어도 해당 모듈만 수정하면 전체 시스템에는 영향이 없습니다.
2. **테스트 가능성 (Testability)**: 전체를 실행하지 않고도 "가격 계산이 정확한지", "인증이 잘 되는지" 개별적으로 검증할 수 있습니다.
3. **재사용성 (Reusability)**: 향후 네이버 스마트스토어나 쿠팡 등으로 확장할 때 기존 인증 및 ERP 모듈을 그대로 가져다 쓸 수 있습니다.
4. **가독성 (Readability)**: 기능별로 코드가 분리되어 있어 협업이나 코드 리뷰 시 로직 파악이 매우 빠릅니다.

## 📁 체계적인 모듈 설계 (System Architecture)

현재 `ERP/` 폴더는 다음과 같은 전문가급 구조로 설계 및 구축되었습니다:

### 1. 핵심 모듈 구성
| 레이어 | 모듈명 | 상태 | 주요 역할 |
| :--- | :--- | :--- | :--- |
| **Configuration** | `config/settings.py` | **[Done]** | 마진율, API 버전, 파일 경로 등 전역 설정 관리 |
| **Authentication** | `core/auth/` | **[Done]** | 카페24 OAuth 2.0 세션 관리 및 자동 토큰 갱신 |
| **ERP Connector** | `core/erp/` | **[Done]** | 이카운트 API 클라이언트 (Zone 조회, 로그인, 품목 추출) |
| **Shop Connector** | `core/shop/` | **[Done]** | 카페24 API 클라이언트 (상품 조회 및 가격 업데이트) |
| **Business Logic** | `logic/calculator.py` | **[Done]** | 판매가격 계산 공식 (`원가 * 마진율`) 구현 |
| **Consumer Frontend** | `web/search_quote/` | **[/]** | 소비지용 실시간 검색 및 견적 의뢰 시스템 (v2.0 고도화) |
| **Shop Conversion** | `web/cart_system/` | **[/]** | 장바구니 연동 및 카페24 구매 결제 시스템 연동 |
| **Orchestrator** | `main.py` | **[/]** | 전체 연동 프로세스 실행 컨트롤 플로우 |

### 2. 프로젝트 디렉토리 구조
```
ERP/
├── config/settings.py      # 중앙 집중 설정
├── core/                   # 백엔드 API 연동 (ERP, Cafe24)
├── logic/                  # 비즈니스 로직 (가격, 필터링)
├── web/                    # 프론트엔드 서비스 모듈
│   ├── search_quote/       # [추가] 실시간 검색 및 견적 시스템
│   └── cart_system/        # [추가] 장바구니 및 구매 전환 시스템
├── main.py                 # 실행 엔트리포인트 (통합 버전)
└── log.md                  # 작업 이력 관리
```

## ⚙️ 상세 구현 사양 (v1: PROD_CD 기반)

### 1. 매칭 및 선별 동기화 전략 (Backend)
- **Key**: 이카운트 `PROD_CD` ↔ 카페24 `자체상품코드` (Primary Key)
- **Selective Sync (Excel-Managed)**: `config/Target_List.csv` 파일을 통해 관리.
  - **구조**: `PROD_CD`, `Sync_YN`, `Memo` 등의 열로 구성.
  - **로직**: `Sync_YN` 열에 **'Y'** 표시가 있는 품목만 선별하여 동기화.
  - **이점**: 엑셀에서 시각적으로 관리하기 편하며, 특정 상품을 리스트에서 지우지 않고 'N'으로 바꾸는 것만으로 노출/동기화 제어가 가능.

### 2. 가격 계산 및 서비스 로직 (Process)
- **공식**: `최종 판매가 = ERP 출고단가 * 마진율(1.3)`
- **단위**: 100원 단위 올림(Ceil) 처리 (예: 13,240원 -> 13,300원)

### 3. 소비자 경험 모듈 (Frontend Strategy)
- **Display Strategy: [안 A] 큐레이션형 (Main Option)**:
  - **Excel Sync Control**: `Target_List.csv`에서 `Sync_YN='Y'`인 품목만 메인 화면 큐레이션 섹션에 노출.
  - **Metadata 활용**: 엑셀에 추가한 '카테고리'나 '태그' 정보를 웹 전시 레이아웃 그룹핑에 활용 가능.
  - **Benefit**: 엑셀 하나로 [백엔드 동기화 여부] + [프론트엔드 노출 여부] + [추가 메타데이터] 정보를 통합 관리.

- **Search & Quote**:
  - `Sheet_데이타 연동`에서 시도했던 **Stale-While-Revalidate** 캐싱 적용.
  - 소비자가 품목별 수량을 입력하면 실시간 견적 총액 계산 및 PDF/메일 전송 연동.

- **Cart & Checkout**:
  - 견적 리스트를 카페24 장바구니 API와 연동하여 실제 결제 페이지로 전환.
  - 리액트(`ProductTable`) 기반의 빠른 필터링과 반응형 UI 적용.

## 🔍 단계별 검증 및 통합 전략 (Step-by-Step Strategy)

사용자의 요청에 따라, 전체 시스템을 한 번에 통합하기 전 다음의 단계별 검증 과정을 거칩니다:

1.  **ERP 모듈 검증**: `ECountClient`를 통해 실제 품목 데이터(원가 포함)가 정상적으로 수집되는지 확인합니다.
2.  **쇼핑몰 모듈 검증**: `Cafe24Client`를 통해 상품 조회 및 토큰 갱신이 정상적으로 작동하는지 확인합니다.
3.  **비즈니스 로직 검증**: 마진율 적용 및 올림 처리가 정확한지 단위 테스트를 수행합니다.
4.  **Dry-Run 통합 테스트**: 실제 카페24에 데이터를 전송하지 않고, 로그로만 업데이트 예정 내역을 출력하여 검토합니다.
5.  **최종 통합 및 자동화**: 모든 검증이 완료된 후 실제 동기화를 수행하고 예외 처리를 강화합니다.
