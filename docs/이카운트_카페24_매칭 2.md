# 이카운트 ↔ 카페24 가격 동기화 매칭 및 구현 전략 v2.2

## 1. 프로젝트 목표
이카운트 ERP 가격 변경 시 카페24 연동 상품만 자동으로 가격 업데이트 (서버리스 구현)

## 2. 카페24 연동 상품 필터링 조건
| 조건 | 필드 | 값 | 의미 |
|------|------|------|------|
| 카페24 연동 지시자 | PROD_CD | (1) 포함 | 카페24 대상 |
| 카페24 연동 지시자 | PROD_DES | ◈ 포함 | 카페24 대상 |
| 상품 구분 | CLASS_CD | 11111 | 재고 (디폴트) |

## 3. 확정 스펙
| 항목 | 디폴트값 | 변경 가능 범위 |
|------|------|------|
| 실행 방식 | Google Apps Script (GAS) | 고정 |
| 동기화 주기 | 60분 | 구글 시트 [설정]에서 변경 |
| 가격 필드 | OUT_PRICE2 | OUT_PRICE1~3 중 선택 |
| 상품 구분 | 11111 (재고) | 11111 / 22222 / 전체 |
| 연동 방식 | Cafe24 Admin API 직접 호출 | 고정 |

## 4. 매핑 전략 확정
### 매핑 키
```
이카운트 PROD_CD ↔ 카페24 product_no (숫자)
예: (1)600LXPF301800N ↔ 1671
```

### 매핑 방법
- 카페24 product_no = URL에서 확인
  `product_no=1671`
- 구글 시트 [매핑테이블]에 수동 또는 자동으로 등록
- 이카운트 쇼핑몰품목코드연결 메뉴는 **주문수집 전용** → 우리 프로젝트와 무관

### 카페24 API 업데이트
```
PUT /api/v2/admin/products/{product_no}
→ price 필드 업데이트
```

## 5. 시스템 아키텍처
```
[GAS Trigger - 60분 주기]
        ↓
[이카운트 API]
→ GetBasicProductsList
→ (1) + ◈ + CLASS_CD=11111 필터링
→ OUT_PRICE2 추출
        ↓
[구글 시트 매핑테이블 조회]
→ PROD_CD → product_no 변환
        ↓
[카페24 API]
→ PUT /api/v2/admin/products/{product_no}
→ 가격 업데이트
        ↓
[실행로그 시트 기록]
→ 성공 / 실패 / SKIP
```
