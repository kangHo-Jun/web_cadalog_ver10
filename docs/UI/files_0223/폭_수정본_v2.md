## 1. 현재 구조 문제

현재 사이드바가 `fixed` 포지션으로 브라우저 뷰포트 기준에 고정되어 있고,  
`Phase1CatalogView.tsx`에서 `ml-40`(160px)으로 억지로 맞춰놓은 **가짜 정렬** 상태.

```
현재 (가짜 정렬):
[사이드바: fixed, 브라우저 기준] + [main: ml-40으로 위치 보정]
→ 900px 컨테이너 적용 시 사이드바만 밖으로 이탈
```

---

## 2. 목표 구조

```
[브라우저: bg-[#f3f3f3]]
  └── [max-w-[900px] mx-auto, relative, h-screen, flex]
        ├── [사이드바: flex-shrink-0, w-40]  ← fixed 제거
        └── [main: flex-1, overflow-hidden]
              └── [리스트: flex-1, overflow-y-auto]
```

---

## 3. 수정 지시사항

### FixedSidebar.tsx
- `position: fixed` 제거
- `h-screen` 유지
- `w-40` 유지
- 나머지 스타일 유지

### Phase1CatalogView.tsx
- 최상위 wrapper에 `max-w-[900px] mx-auto flex h-screen relative` 적용
- `ml-40` 오프셋 제거
- 사이드바와 main이 flex로 나란히 배치되도록 구성
