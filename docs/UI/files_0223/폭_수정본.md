# UI 개선 제안서 — 가로 폭 과다 확장 문제 (수정본)

프로젝트: web-cadalog-ver10  
작성일: 2026-02-20  
문제 유형: PC 화면에서 가로 폭이 전체로 펼쳐지는 현상

---

## 1. 현재 문제 정의

PC 환경에서 브라우저 전체 폭을 그대로 사용하고 있어  
콘텐츠가 좌우 끝까지 확장되고 시선이 분산되는 현상 발생.

건축자재 B2B 카탈로그 특성상 **"빠르게 상품을 찾고 담는 것"** 이 최우선이므로  
시선 이동 거리를 최소화하는 구조가 필요하다.

---

## 2. 개선 방향 (의도 명확화)

> **목표: 화려한 UI가 아닌, 빠르고 편하게 상품을 담는 구조**

- 콘텐츠 최대 폭 제한 → 시선 집중
- 중앙 정렬 구조
- **1열 리스트 유지** (그리드 전환 없음)
- 좌우 여백으로 카탈로그 프레임 강화

---

## 3. 개선 전략

### 3-1. 최대 폭 제한 ✅

```tsx
<div className="max-w-[900px] mx-auto">
```

> ⚠️ 원본 문서의 1200px은 과도하게 넓음.  
> 900px이 시선 이동 최소화 + 카탈로그 집중도에 최적.  
> 정확한 수치는 F12 개발자 도구에서 직접 보면서 확정할 것.  
> 후보: `800px → 900px → 1000px`

### 3-2. 레이아웃 유지 ✅

```tsx
// 기존 사이드바 + 1열 리스트 구조 그대로 유지
// Grid 2~3열 전환 금지
```

> **그리드 전환 하지 않는 이유:**  
> 1열 리스트는 눈이 위아래로만 움직여 담기 버튼 도달이 가장 빠름.  
> 그리드는 눈이 좌우로도 움직여야 해서 "빠르게 담기" 목적과 반대.

### 3-3. 좌우 배경 분리 ✅

```tsx
<div className="min-h-screen bg-[#f3f3f3]">
  <div className="max-w-[900px] mx-auto bg-white shadow-sm relative">
    {/* 사이드바 + 리스트 */}
  </div>
</div>
```

> ⚠️ `position: relative` 필수 — 바텀 시트 absolute 기준점으로 사용

---

## 4. 최종 구조 목표

```
[브라우저 전체: bg-[#f3f3f3]]
  └── [max-w-[900px] 중앙 컨테이너: relative, shadow-sm]
        ├── 사이드바 (카테고리 고정)
        └── 메인 영역
              ├── Header
              └── Product List (1열, flex-1, overflow-y-auto)
```
