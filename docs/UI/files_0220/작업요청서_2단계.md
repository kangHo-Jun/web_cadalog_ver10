# 작업 요청서 — 2단계: 레이아웃 너비 제한

> 우선순위: 높음 | 난이도: ★★☆☆☆  
> 수정 파일: `Phase1CatalogView.tsx`, `StickySearchHeader.tsx`, `FixedSidebar.tsx`

---

## 문제

브라우저 전체 너비를 사용해서 콘텐츠가 지나치게 넓게 퍼져 보임.  
특히 와이드 모니터에서 상품 정보가 좌우로 너무 분산되어 가독성 저하.

## 요구사항

- 전체 레이아웃을 중앙 정렬로 제한
- 양옆 외부 배경은 `#f3f3f3` 유지
- 사이드바는 레이아웃 안에 포함되어야 함 (이탈 금지)
- 1단계 작업 결과(스크롤 고정) 유지

## 수정 원칙

```
외부 배경:     min-h-screen bg-[#f3f3f3]
중앙 컨테이너: max-w-[900px] mx-auto w-full h-screen
               shadow-2xl bg-white
               position: relative  ← 3단계 바텀 시트를 위해 필수
내부 구조:     사이드바(고정) + 메인영역(flex-1) 유지
```

> ⚠️ `position: relative` 는 3단계 바텀 시트 작업을 위해 반드시 지금 추가할 것

## 완료 기준

- [ ] 전체 레이아웃이 최대 900px로 중앙 정렬됨
- [ ] 양옆 배경색 `#f3f3f3` 표시됨
- [ ] 사이드바가 컨테이너 안에 포함되어 이탈하지 않음
- [ ] 1단계 스크롤 고정 동작 유지
- [ ] 컨테이너에 `position: relative` 적용 확인
