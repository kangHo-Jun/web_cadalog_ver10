## 4. 대표 상품 테이블 (CRITICAL)

### 정의
- 각재/목재 선택 시 **옵션이 아닌 “대표 상품 목록”만 테이블 형태로 표시**한다.
- 대표 상품은 옵션 묶음의 상위 개념이다.

### 규칙
- 테이블 **행(row) 클릭만** 옵션 Overlay 진입 트리거가 된다.
- 버튼/입력 요소 클릭은 기존 동작을 유지한다.

---

## 5. 옵션 Glass Overlay (CRITICAL)

### 역할
- 대표 상품에 속한 **옵션 상품을 나열**
- Overlay 내부에서 다음을 **한 번에 처리**한다:
  - 규격 선택
  - 수량 입력
  - 견적 담기

### 제한
- Overlay 밖으로 나가야 견적 담기가 가능한 UX ❌
- 가격 계산/로직 수정 ❌

---

## 6. Glass Overlay 시각 규칙

- 배경: `backdrop-filter: blur`
- 콘텐츠 영역:
  - 불투명 (bg-white 또는 동등)
  - 숫자/규격 가독성 최우선
- 페이지 이동 ❌

## 10. 적용 범위 및 확장 규칙 (MANDATORY)

- **각재/목재는 검증용 기준 카테고리**이다.
- 각재/목재에서 기준 테스트를 통과하면,
  **모든 다른 품목 카테고리에 동일한 UI/UX 구조를 적용해야 한다.**

### 10.2 금지 사항
- 카테고리별 전용 UI ❌

### 10.3 성공 기준
- 모든 품목에서
  **대표 상품 테이블 → 옵션 Glass Overlay → 견적 담기**
  흐름이 동일해야 한다.
- 기존 Baseline 결과는 유지되어야 한다.

## 12. 기준 테스트 (Baseline)

- 기준 URL: `/quote?root=327&sub=332`
- 기준 파일: `baseline_v1_quote_root327_sub332.md`
- 상품 목록/가격/합계 결과 동일해야 함


대표 상품과 옵션 상품은
동일한 상품 그룹 식별자(카페24 상품코드 prefix 또는 공통 그룹 키)를 기준으로 연결된다.

옵션 Overlay의 견적 담기 트리거는
모든 카테고리에서 ‘견적’ 문자열을 포함한 텍스트 또는 aria-label을 가져야 한다.
