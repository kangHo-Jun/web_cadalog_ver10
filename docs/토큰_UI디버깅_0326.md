## 9. Redis 오류가 발생할 수 있는 유일한 경우

현재 자동화 시스템으로 대부분의 Redis 오류는 자동 복구됩니다.
단, 아래 경우는 수동 대응이 필요합니다.
refresh_token 만료 (14일) 알림 이메일을 놓쳤을 때
→ Redis 토큰 만료
→ 환경변수 토큰도 만료
→ 자동 갱신 불가
→ 서비스 중단 (401 오류)

### 이 경우 해결 방법
6번 수동 재인증 절차 즉시 실행
python3 scripts/reauth_cafe24.py

### 예방 방법
✅ 14일마다 수신되는 이메일 알림 반드시 확인
✅ 알림 수신 즉시 재인증 실행 (미루지 말 것)
✅ 구글시트 모니터링 주기적 확인

## 10. 추가 발생 오류 및 해결 (2026-03-13)

### 오류: Uncaught TypeError: Cannot read properties of undefined (reading 'length')

**증상:**
- 사이트 접속 시 Application error 발생
- 브라우저 콘솔: Cannot read properties of undefined (reading 'length')

**원인:**
- debug-snapshot API 응답 구조 변경
- 변경 전: data = { P0000CAT: {...}, ... } (스냅샷 직접 반환)
- 변경 후: data = { status:'OK', responseTime:'...', lastSnapshot: {...} }
- 프론트엔드는 data를 바로 Object.values()로 사용하고 있었음
- data.lastSnapshot이 아닌 data 자체를 파싱 시도 → undefined → .length 에러

**해결:**
- 파일: src/components/phase1/Phase1CatalogView.tsx
- 변경 전: Object.values(data)
- 변경 후: Object.values(data.lastSnapshot ?? data)

**교훈:**
- API 응답 구조 변경 시 프론트엔드도 반드시 함께 수정
- debug-snapshot 같이 여러 곳에서 사용하는 API는
  구조 변경 시 영향 범위를 먼저 파악할 것
