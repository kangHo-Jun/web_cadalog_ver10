# **1\. 프로젝트 개요**

이카운트 ERP에서 상품 가격이 변경될 때, 카페24 쇼핑몰의 판매가를 자동으로 동기화하는 Python 자동화 스크립트를 개발합니다.

| 항목 | 내용 |
| :---- | :---- |
| 프로젝트명 | 이카운트-카페24 가격 자동 동기화 |
| 개발 언어 | Python 3.10 이상 |
| 실행 방식 | 5분 주기 자동 스케줄링 (cron 또는 APScheduler) |
| 핵심 목표 | 이카운트 가격 변경 감지 → 카페24 즉시 업데이트 |
| 배포 환경 | 로컬 서버 또는 클라우드 (AWS/GCP/NCP 모두 가능) |

# **2\. 기술적 배경 및 제약사항**

## **2-1. 이카운트 Webhook 미지원 이유**

이카운트 OAPI는 데이터 변경 시 외부로 알림을 보내는 Webhook 기능을 공식 지원하지 않습니다. 따라서 '진짜 실시간' 감지는 불가능하며, 폴링(Polling) 방식으로 구현합니다.

## **2-2. 폴링(Polling) 방식 설명**

* 5분마다 이카운트에서 전체 상품 가격 목록을 가져옴

* 직전 실행 시 저장해둔 가격과 비교

* 변경된 상품만 카페24 API로 가격 업데이트

* 실제 체감상 5분 이내 동기화 \= 사실상 준실시간

| POST /OAPI/V2/InventoryBasic/GetBasicProductsList?SESSION\_ID={SESSION\_ID} Request Body: {   "PAGE\_SIZE": 100,       // 한 번에 가져올 상품 수 (최대 100\)   "PAGE\_CURRENT": 1,      // 페이지 번호 (페이지네이션)   "PROD\_TYPE": "",        // 품목구분 (공백=전체)   "USE\_FLAG": "Y"         // 사용 중인 상품만 } Response: {   "Data": {     "TotalCnt": 250,     "Result": \[       { "PROD\_CD": "ITEM001", "PROD\_DES": "상품명", "OUT\_PRICE": 15000 },       ...     \]   } } |
| :---- |

## 4-4. 카페24 API 연동 (cafe24_api.py) — 확정 전략

### 매핑 전략 (매핑 CSV 불필요 — 완전 자동화)
이카운트 PROD_CD → 카페24 product_code 검색 → product_no 획득 → 가격 업데이트

### STEP 1. 카페24 상품 검색
GET /api/v2/admin/products?product_code={PROD_CD}
Host: daesan3833.cafe24api.com
Authorization: Bearer {ACCESS_TOKEN}

→ 결과 없으면 SKIP (카페24 미연동 상품)
→ 결과 있으면 product_no 추출 후 STEP 2

### STEP 2. 가격 업데이트
PUT /api/v2/admin/products/{product_no}
Host: daesan3833.cafe24api.com
Authorization: Bearer {ACCESS_TOKEN}

Request Body:
{
  "shop_no": 1,
  "request": {
    "price": "변경가격"
  }
}

### 수정된 이카운트 상품조회 API (확정)
- 엔드포인트: InventoryBasic/GetBasicProductsList  ← (GetListGoods 아님)
- 가격 필드: OUT_PRICE  ← (retail_price 아님)

### T5 검증 기준
- 카페24 update_price() → HTTP 200 응답
- PROD_CD 없는 상품 → SKIP 처리 확인

## **4-5. 가격 비교 로직 (sync\_engine.py)**

| \# 핵심 로직 의사코드 def sync\_prices():     \# 1\. 이카운트에서 현재 가격 가져오기     current\_prices \= ecount.get\_all\_products()       \# { 'ITEM001': 15000, 'ITEM002': 8000, ... }     \# 2\. 캐시(저장된 이전 가격)와 비교     cached\_prices \= load\_cache('price\_cache.json')     \# 3\. 변경된 상품만 추출     changed \= \[\]     for prod\_cd, new\_price in current\_prices.items():         if cached\_prices.get(prod\_cd) \!= new\_price:             changed.append((prod\_cd, new\_price))     \# 4\. 변경된 상품만 카페24 업데이트     for prod\_cd, new\_price in changed:         cafe24\_product\_no \= get\_mapping(prod\_cd)         cafe24.update\_price(cafe24\_product\_no, new\_price)         log(f'{prod\_cd}: {old\_price} → {new\_price}')     \# 5\. 캐시 업데이트     save\_cache(current\_prices) |
| :---- |

## **4-6. 상품 코드 매핑 자동화**

이카운트 품목코드(PROD_CD)와 카페24 상품번호(product_no)는 별도의 매핑 CSV 파일 없이, 카페24 상품 검색 API(`GET /api/v2/admin/products?product_code={PROD_CD}`)를 통해 실시간으로 매핑됩니다.

- 이카운트 `PROD_CD`가 카페24의 `product_code`에 동일하게 등록되어 있어야 합니다.
- 카페24에 해당 코드가 없는 상품은 자동으로 건너뜁니다(SKIP).
