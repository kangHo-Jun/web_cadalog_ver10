# 토큰/연결 문제 정리

이 문서는 Cafe24 API 토큰 및 네트워크 연결 문제를 겪으면서 확인된 증상과 해결 흐름을 정리한 것입니다.

## 1) 주요 증상 요약

- **DNS 실패**: `Could not resolve host` 오류 발생
- **연결 차단**: `Operation not permitted`로 443 연결 실패
- **토큰 오류**: `invalid_grant / Invalid client_secret` 응답
- **API 버전 오류**: `2024-03-01 version you requested is not available`
- **401 Unauthorized**: 토큰 만료/무효 또는 잘못된 버전/요청

## 2) 원인별 해결 흐름

### A. DNS/호스트 문제

1. `/etc/hosts`에 **잘못된 IP**가 등록되면 강제 해석되어 접속 실패
2. 잘못된 항목 제거 후 DNS 캐시 플러시 필요
3. 올바른 IP 확인은 `nslookup`로 확인 후 적용

권장 순서:

```
sudo sed -i '' '/daesan3833.cafe24api.com/d' /etc/hosts
sudo dscacheutil -flushcache
nslookup daesan3833.cafe24api.com 8.8.8.8
```

필요 시 확인된 IP로 재등록:

```
sudo sh -c 'echo "IP_ADDR daesan3833.cafe24api.com" >> /etc/hosts'
sudo dscacheutil -flushcache
```

### B. 네트워크/방화벽 차단

- **HTTPS 연결이 즉시 실패**하면 로컬 보안 정책/방화벽 차단 가능성이 높음
- 핫스팟 연결을 해도 동일 증상이면 **보안 프로그램/방화벽**을 먼저 확인

확인 명령:

```
curl -I -m 10 https://daesan3833.cafe24api.com | head -n 5
```

### C. 토큰/Client Secret 문제

오류 예시:
- `invalid_grant` + `Invalid client_secret`

해결:
1) `config/secrets.env`의 `CAFE24_CLIENT_SECRET` 값 확인  
2) 새 code 발급 후 토큰 재발급 진행  

### D. API 버전 문제

오류 예시:
- `2024-03-01 version you requested is not available. The default value for the app version is 2025-12-01.`

해결:
- `X-Cafe24-Api-Version: 2025-12-01`로 설정

## 3) 정상 동작 확인 절차 (최소 경로)

1) TLS 연결 확인
```
curl -I -m 10 https://daesan3833.cafe24api.com | head -n 5
```

2) 토큰 재발급 (code 필요)

```
POST https://{mall_id}.cafe24api.com/api/v2/oauth/token
grant_type=authorization_code
code=...
redirect_uri=...
```

3) 상품 조회 (버전 2025-12-01 고정)

```
GET /api/v2/admin/products
Header: X-Cafe24-Api-Version: 2025-12-01
```

## 4) 최종 정상 상태

- `X-Cafe24-Api-Version: 2025-12-01` 적용 시 정상 응답 확인
- Cafe24 상품 총 898건 조회 및 JSON 저장 완료

## 5) 체크리스트

- [ ] `/etc/hosts`에 잘못된 cafe24api 도메인 IP가 없음
- [ ] 방화벽/보안SW에서 443 차단 없음
- [ ] `CAFE24_CLIENT_SECRET` 최신 값 적용
- [ ] API 버전 `2025-12-01` 적용
- [ ] 토큰 만료/갱신 확인
