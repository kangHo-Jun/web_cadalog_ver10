# 카페24 가격 자동 동기화 — 시스템 가이드

> 파일: `scripts/p7_gas.js` (GAS 배포본: `gas-push/Code.js`)
> 최종 업데이트: 2026-03-25

---

## 1. 한눈에 보는 전체 흐름

이 시스템은 **이카운트(ERP)의 상품 가격**을 **카페24(쇼핑몰)에 자동으로 반영**합니다.
Google Apps Script(GAS)가 매 시간마다 자동으로 실행되며, 사람이 직접 개입할 필요가 없습니다.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  매 1시간마다 자동 실행
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 STEP 1  buildCafe24Cache()
         카페24 쇼핑몰에서 현재 상품 목록 + 가격을 다운로드
         → [카페24상품] 시트에 저장

         ↓ 완료 후 10분 대기 (one-time 트리거 예약)

 STEP 2  syncPrices()
         이카운트에서 현재 가격 조회
         [카페24상품] 시트의 가격과 비교
         → 가격이 바뀐 항목만 카페24에 PUT (업데이트)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  별도 1시간마다 자동 실행 (monitoring-gas 스프레드시트)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 TOKEN   refreshCafe24Token()  (monitoring-gas)
         카페24 access_token 갱신
         → 모니터링 시트 [설정] 탭 저장
         → Vercel /api/token-update → Redis 업데이트
```

## 2. 자동 실행 주기

| 작업 | 실행 주기 | 담당 함수 | 스프레드시트 |
|------|-----------|-----------|-------------|
| 카페24 상품 다운로드 | **1시간마다** (고정 트리거) | `buildCafe24Cache()` | p7_gas (동기화 시트) |
| 가격 비교 & 업데이트 | buildCafe24Cache 완료 **10분 후** (one-time 트리거) | `syncPrices()` | p7_gas (동기화 시트) |
| access_token 갱신 | **1시간마다** (고정 트리거) | `refreshCafe24Token()` | monitoring-gas (모니터링 시트) |
| refresh_token 갱신 | **14일마다** (수동) | 카페24 재인증 URL 사용 | — |

> **왜 10분 간격인가?**
> `buildCafe24Cache`가 카페24에서 전체 상품을 다운로드하는 데 수 분이 걸립니다.
> 다운로드가 완전히 끝난 뒤에 가격 비교를 해야 정확하므로, 완료 직후 10분 딜레이를 두고 `syncPrices`를 예약합니다.

## 3. STEP 1: 카페24 상품 다운로드 — buildCafe24Cache()

### 무엇을 하는가?

카페24 쇼핑몰에 등록된 **판매 중인 상품 전체**를 조회하여 구글 시트에 저장합니다.
특히 각 옵션(variant)의 **자체 상품코드(custom_variant_code)** 와 **현재 가격(additional_amount)** 을 저장합니다.

### 동작 순서

1. 카페24 API — 판매 중 상품 목록을 100건씩 페이지 단위로 조회
2. 각 상품의 옵션(variants) 개별 조회
3. `custom_variant_code`(자체 상품코드)가 입력된 옵션만 필터링
4. [카페24상품] 시트에 기존 데이터와 비교하여 신규/삭제 항목만 반영 (증분 업데이트)
5. 완료 후 → `syncPrices` 10분 후 one-time 트리거 예약

### 저장되는 정보: [카페24상품] 시트

| 열 | 필드명 | 내용 | 예시 |
|----|--------|------|------|
| A | product_no | 카페24 상품 번호 | 1870 |
| B | product_code | 카페24 상품 코드 | P0000CTY |
| C | product_name | 상품명 | 마대(80L) |
| D | custom_variant_code | **이카운트 PROD_CD** (매칭 키) | (1)800마대80 |
| E | variant_code | 카페24 옵션 코드 (가격 업데이트 시 사용) | P0000CTY000A |
| F | additional_amount | **카페24 현재 가격** (VAT 포함) | 13500 |

> `custom_variant_code`는 카페24 관리자에서 수동 입력해야 합니다.
> 카페24 관리자 → 상품 → 옵션 탭 → **자체 상품코드(옵션)** 란에 이카운트 PROD_CD를 입력.

### 관련 API 호출

```
GET https://{mallId}.cafe24api.com/api/v2/admin/products
  ?selling=T&limit=100&offset={N}&fields=product_no,product_code,product_name

GET https://{mallId}.cafe24api.com/api/v2/admin/products/{product_no}/variants
  ?fields=variant_code,custom_variant_code,additional_amount
```

## 4. STEP 2: 가격 비교 & 업데이트 — syncPrices()

### 무엇을 하는가?

이카운트의 현재 가격과 카페24의 현재 가격을 비교하여, **가격이 변동된 항목만** 카페24에 업데이트합니다.
변동이 없는 항목은 API 호출하지 않으므로 불필요한 요청이 없습니다.

### 가격 비교 방법

```
이카운트 가격 (OUT_PRICE2)
    │
    ▼
VAT 10% 포함 계산
ecount_price_vat = Math.round(이카운트 가격 × 1.1)
    │
    ▼
[카페24상품] 시트의 additional_amount 와 비교
    │
    ├─ 같으면 → 스킵 (API 호출 없음)
    └─ 다르면 → 카페24 PUT 요청 (가격 업데이트)
```

> **예시**: 이카운트 가격 18,000원 → VAT 포함 19,800원 → 카페24 `additional_amount` 가 19,800이 아니면 업데이트

### 이카운트 필터링 조건

모든 이카운트 상품을 동기화하지 않습니다. 아래 두 조건을 모두 만족하는 상품만 대상:

```javascript
PROD_CD에 '(1)' 포함    // 다산3833 제품 코드 규칙
PROD_DES에 '◈' 포함    // 카탈로그 등록 대상 품목 표시
```

### 매칭 방법

두 시스템의 상품을 연결하는 키:

| 시스템 | 필드 | 예시 |
|--------|------|------|
| 이카운트 | PROD_CD | `(1)800마대80` |
| 카페24 | custom_variant_code | `(1)800마대80` |

두 값이 **정확히 일치**하면 같은 상품으로 인식합니다.

### 카페24 가격 업데이트 API

```
PUT https://{mallId}.cafe24api.com/api/v2/admin/products/{product_no}/variants/{variant_code}
Body: { "shop_no": 1, "request": { "additional_amount": "19800" } }
```

- 옵션 없는 단품: `PUT /products/{product_no}` 직접 호출

### 결과 기록: [매핑테이블] 시트

실행 후 모든 처리 결과가 이 시트에 기록됩니다.

| 열 | 필드명 | 내용 |
|----|--------|------|
| A | custom_variant_code | 이카운트 PROD_CD |
| B | product_no | 카페24 상품번호 |
| C | variant_code | 카페24 옵션 코드 |
| D | ecount_price_vat | 이카운트 가격 × 1.1 (VAT 포함) |
| E | 최근업데이트 | 처리 시각 |
| F | 결과 | 성공 / 스킵(변동없음) / 미매핑 / 실패(상태코드) |

## 5. 토큰 관리

카페24 API를 호출하려면 **access_token** 이 필요합니다. 이 토큰은 2시간마다 만료되므로 자동 갱신이 필요합니다.

### 토큰 흐름

```
monitoring-gas 스프레드시트
    │
    ├─ refreshCafe24Token() — 1시간마다 자동 실행
    │   1. 카페24 API에 refresh_token으로 새 access_token 요청
    │   2. 모니터링 시트 [설정] 탭에 저장
    │       - CAFE24_ACCESS_TOKEN
    │       - CAFE24_REFRESH_TOKEN
    │       - TOKEN_EXPIRES_AT
    │       - REFRESH_EXPIRES_AT
    │   3. Vercel /api/token-update 호출 → Redis 업데이트
    │
    └─ p7_gas (동기화 시트)
        buildCafe24Cache / syncPrices 실행 시
        → initMonitoringSheet_() 로 모니터링 시트에서 토큰 자동 로드
```

### 토큰 만료 기간

| 토큰 종류 | 유효 기간 | 갱신 방법 |
|-----------|-----------|-----------|
| access_token | 2시간 | 자동 (1시간마다 갱신) |
| refresh_token | 14일 | 수동 재인증 (만료 1일 전 이메일 알림 발송) |

> **refresh_token이 만료되면** access_token 자동 갱신이 불가능해집니다.
> 만료 1일 전 이메일 알림이 발송되므로, 알림 수신 시 즉시 재인증 필요.
> 재인증: 모니터링 시트 메뉴 → 🛠️ 모니터링 → 토큰 재인증 URL

## 6. 미매핑 상품 처리

`syncPrices` 실행 후 [매핑테이블] F열에 **미매핑** 으로 표시된 항목이 있다면:

**원인**: 카페24 옵션의 자체 상품코드가 비어 있어 이카운트 PROD_CD와 연결되지 않은 상태

**해결 절차**:
1. [매핑테이블] 에서 미매핑 항목의 PROD_CD 확인
2. 카페24 관리자 → 해당 상품 → 옵션 탭 → **자체 상품코드(옵션)** 에 PROD_CD 입력
3. GAS 편집기에서 `buildCafe24Cache()` 수동 실행 (또는 다음 1시간 트리거 대기)
4. 이후 `syncPrices()` 실행 시 자동으로 매핑되어 가격 업데이트

## 7. 최초 설정 / 트리거 초기화

처음 설치하거나 트리거가 사라진 경우:

1. GAS 편집기 → 함수 선택 `createTrigger` → 실행
2. 설정되는 트리거:
   - `autoRefreshCafe24Token` — 1시간마다 (동기화 시트)
   - `buildCafe24Cache` — 1시간마다 (동기화 시트)
   - `syncPrices` — 고정 트리거 없음 (`buildCafe24Cache` 완료 후 자동 예약)

> **초기화 (전체 재다운로드)** 가 필요한 경우:
> GAS 편집기 → 함수 선택 `initCafe24Cache` → 실행
> 카페24 상품 전체를 처음부터 다시 다운로드하고, 완료 후 자동으로 `syncPrices` 예약
