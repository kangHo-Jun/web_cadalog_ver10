# 웹카탈로그 시스템 가이드

> 프로젝트: `Next.js` (Vercel 배포)
> 배포 URL: `https://web-cadalog-ver10.vercel.app`
> 최종 업데이트: 2026-03-25

---

## 1. 프로젝트 목표

카페24 쇼핑몰(daesan3833)에 등록된 상품과 가격을 **웹카탈로그**로 표시하고, 거래처에게 **견적서**를 발행하는 시스템.

| 기능 | 설명 |
|------|------|
| 웹카탈로그 | 카페24 상품을 카테고리별로 표시, VAT 별도 공급가 표시 |
| 견적서 | 상품 선택 → 수량 입력 → PDF 출력 |
| 가격 자동 동기화 | 카페24 `additional_amount` 를 매일 자동 반영 |
| 관리자 즉시 동기화 | 버튼 1클릭으로 즉시 가격 갱신 |

## 2. 전체 시스템 아키텍처

```
┌────────────────────────────────────────────────────────────┐
│  카페24 (daesan3833.cafe24api.com)                          │
│  - 상품 정보 (product_no, product_name, price)              │
│  - 옵션 가격 (additional_amount) ← GAS syncPrices() 로 갱신 │
└────────────────────┬───────────────────────────────────────┘
                     │ Cafe24 Admin API (Bearer Token)
                     ▼
┌────────────────────────────────────────────────────────────┐
│  Vercel (Next.js)                                          │
│                                                            │
│  [Cron] /api/cron/sync-products  ─ 매일 03:00 UTC (12:00 KST)
│  [수동] POST /api/sync-products  ← 관리자 페이지 버튼       │
│                                                            │
│  1. 카테고리별 상품 + 옵션 조회                               │
│  2. additional_amount 기반 가격 구성                         │
│  3. Redis에 스냅샷 저장 (catalog:snapshot:v1)               │
└────────────────────┬───────────────────────────────────────┘
                     │ Redis (Upstash/RedisLabs)
                     ▼
┌────────────────────────────────────────────────────────────┐
│  Redis                                                     │
│  - catalog:snapshot:v1  → 전체 상품 JSON (TTL 없음)         │
│  - cafe24_tokens        → access_token / refresh_token     │
└────────────────────┬───────────────────────────────────────┘
                     │ SWR /api/debug-snapshot
                     ▼
┌────────────────────────────────────────────────────────────┐
│  브라우저 (Phase1CatalogView, QuoteCatalog)                 │
│  - 상품 목록 표시                                            │
│  - VAT 제외 공급가 표시 (/1.1)                              │
│  - 장바구니 → 견적서 PDF                                    │
└────────────────────────────────────────────────────────────┘
```

### 토큰 관리 흐름 (별도)

```
monitoring-gas (Google Apps Script)
    │
    ├─ refreshCafe24Token()  1시간마다 자동 실행
    │   1. 카페24 API → 새 access_token 발급
    │   2. 모니터링 구글시트 [설정] 저장
    │   3. POST /api/token-update → Redis 업데이트
    │
    └─ Redis cafe24_tokens 키 갱신 완료
         → apiClient.ts interceptor가 Bearer 토큰으로 자동 사용
```

## 3. 가격 표시 로직

### 카페24 가격 구조

카페24에서 가격은 두 가지로 나뉩니다:

| 필드 | 의미 | 사용 여부 |
|------|------|-----------|
| `price` | 상품 기본가격 (VAT 포함) | 단일 상품(옵션 없음)에 사용 |
| `additional_amount` | 옵션별 추가금액 (VAT 포함) | 옵션 상품에 사용 |

옵션 상품의 최종 가격:
```
최종 가격 = price (기본가) + additional_amount (옵션 추가금)
```

> **예시**: 마대(80L) — price=0, additional_amount=13,500 → 최종 13,500원 (VAT 포함)

### VAT 제외 처리

카페24 저장 가격은 **VAT 포함가**입니다. 화면에는 **VAT 별도 공급가**로 표시합니다.

```typescript
// src/lib/price-utils.ts
export const toSupplyPrice = (vatIncluded: number): number =>
    Math.round(vatIncluded / 1.1);
```

| 저장값 (카페24) | 표시값 (웹카탈로그) | 비고 |
|----------------|-------------------|------|
| 13,500원 | 12,273원 | 공급가 (VAT 별도) |
| 19,800원 | 18,000원 | 공급가 (VAT 별도) |

### 가격 표시 위치별 처리

| 컴포넌트 | 처리 방식 | 표시 문구 |
|---------|---------|---------|
| Phase1CatalogView | VAT 포함가 그대로 | `(VAT 포함)` |
| ProductListPhase1 | `formatSupplyPrice()` → /1.1 | `(VAT 별도)` |
| CartDrawer | `toSupplyPrice()` → /1.1 | 공급가 |
| 견적서 summary | `toSupplyPrice()` → /1.1 | 공급가 |

## 4. 데이터 흐름 상세

### 동기화 단계별 흐름

```
① GAS syncPrices()  (1시간 주기)
   이카운트 가격 × 1.1 → 카페24 additional_amount PUT

② Vercel Cron  (매일 03:00 UTC / 한국 12:00)
   /api/cron/sync-products GET 호출
       │
       ├─ apiClient → 카테고리 [192,25,50,26,203,197,223,349,350] 순회
       ├─ embed=options,variants → additional_amount 포함 조회
       ├─ 상품 구조 정규화 (GroupedProduct)
       │     parentCode → { parentName, children: [{ name, price, variantCode }] }
       └─ Redis catalog:snapshot:v1 저장

③ 브라우저 요청 시
   SWR → /api/debug-snapshot → Redis 읽기 → 상품 JSON 반환
   → Phase1CatalogView 렌더링
```

### Redis 키 목록

| 키 | 내용 | TTL |
|----|------|-----|
| `catalog:snapshot:v1` | 전체 상품 스냅샷 JSON | 없음 (영구) |
| `cafe24_tokens` | access_token / refresh_token | 없음 (영구) |

## 5. 자동 동기화 트리거 주기

| 작업 | 주기 | 담당 | 비고 |
|------|------|------|------|
| **카탈로그 스냅샷 갱신** | 매일 1회 (03:00 UTC) | `vercel.json` cron | Redis snapshot 업데이트 |
| **access_token 갱신** | 1시간마다 | monitoring-gas `refreshCafe24Token()` | Redis cafe24_tokens 업데이트 |
| **수동 즉시 동기화** | 버튼 클릭 시 | 관리자 페이지 `/admin` | POST /api/sync-products |

`vercel.json` Cron 설정:
```json
{
  "crons": [
    { "path": "/api/cron/sync-products", "schedule": "0 3 * * *" }
  ]
}
```

## 5-1. 웹카탈로그 트리거 지침 (운영자용)

아래 순서대로 이해하면 전체 자동화 흐름이 한 번에 정리됩니다.

**A. GAS (Google Sheets)**
- **정기 동기화**
  - `buildCafe24Cache()` **1시간마다** 실행 (카페24 상품/옵션 캐시 갱신)
  - `syncPrices()`는 `buildCafe24Cache()` 완료 후 **10분 뒤 one-time 트리거**로 실행
  - `refreshCafe24Token()`은 **monitoring-gas에서 1시간마다** 실행
- **수동 즉시 동기화**
  - 구글시트 메뉴: `▶ 즉시 동기화 (카페24 다운로드 + 가격 업데이트)`
  - 실행 함수: `buildCafe24Cache()` → 완료 후 `syncPrices()` 예약

**B. Vercel (서버/크론)**
- **카탈로그 스냅샷 생성**
  - `/api/cron/sync-products` **매일 03:00 UTC (KST 12:00)** 실행
  - 결과는 Redis `catalog:snapshot:v1` 저장
- **가격 스냅샷 생성**
  - `/api/cron/snapshot-prices` **매일 15:00 UTC (KST 00:00)** 실행

**C. 모니터링**
- monitoring-gas `checkAll()` **6시간마다** 실행
- Redis 상태 확인은 `/api/debug-snapshot` 호출로 판별

## 6. 관리자 페이지 사용법

**URL**: `https://web-cadalog-ver10.vercel.app/admin`

### 기능

- **가격 즉시 동기화** 버튼: 클릭 즉시 카페24 최신 가격을 조회하여 Redis를 갱신
- 동기화 완료 후 상품 건수 표시
- 마지막 수동 동기화 시각 표시

### 사용 시나리오

```
카페24 상품 가격 변경
    │
    ├─ 자동 처리:
    │   1. GAS syncPrices() — 카페24 additional_amount 업데이트 (1시간 내)
    │   2. Vercel Cron — 오늘 12:00(KST)에 Redis 자동 반영
    │
    └─ 즉시 반영 필요 시:
        1. /admin 접속
        2. "가격 즉시 동기화" 버튼 클릭 (약 10~30초 소요)
        3. "완료 — 상품 N건 업데이트됨" 확인
        4. 웹카탈로그 새로고침 → 최신 가격 표시
```

## 9. 로컬 개발 환경 설정

```bash
# 1. 의존성 설치
npm install

# 2. 환경변수 설정
cp .env.local.backup .env.local
# .env.local에 위 환경변수 목록 입력

# 3. 개발 서버 실행
npm run dev
# → http://localhost:3000
```

> **주의**: `CAFE24_ACCESS_TOKEN`은 2시간마다 만료됩니다.
> 로컬 개발 시 만료되면 monitoring-gas 모니터링 시트에서 최신 토큰을 복사하여 `.env.local`에 붙여넣으세요.
