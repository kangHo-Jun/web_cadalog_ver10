# 인천창고 재고현황 자동화 개발문서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목표 | 인천창고(WH_CD: 200) 실제 재고 자동 산출 및 Google Sheets 기록 |
| 갱신 주기 | 매 1시간 |
| 실행 환경 | Google Apps Script (별도 신규 프로젝트) |
| 스프레드시트 | 신규 생성 |

---

## 2. 핵심 제약사항 (개발 전 확인 완료)

| 항목 | 결과 |
|------|------|
| 이카운트 재고현황 조회 API | ✅ 사용 가능 (`GetListInventoryBalanceStatus`) |
| 이카운트 판매조회 API | ❌ 미제공 (공식 확인) |
| 이카운트 출고조회 API | ❌ 미제공 (실호출 확인) |
| 이카운트 회계조회 API | ❌ 미제공 (실호출 확인) |
| 창고별 재고 분리 조회 | ❌ WH_CD 지정해도 전체 합산값만 반환 |

> **결론:** 이카운트 API로 판매/출고 내역 조회 불가. 조기출고 처리는 Google Sheets 별도 입력으로 최소화.

---

## 3. 재고 계산 로직

```
실제재고 = 장부재고(BAL_QTY) - 조기출고수량(당일 누적)
```

- **장부재고**: 이카운트 API `GetListInventoryBalanceStatus` 호출값
- **조기출고수량**: 담당자가 Google Sheets 조기출고 시트에 직접 입력
- **자정 초기화**: 조기출고 시트는 매일 00:00 자동 초기화

---

## 4. Google Sheets 구조

### 4-1. 시트 구성 (총 2개 시트)

| 시트명 | 용도 |
|--------|------|
| `재고현황` | 이카운트 API 자동 갱신 + 실제재고 계산 결과 |
| `조기출고` | 담당자 수동 입력 (조기 출고 발생 시만) |

---

### 4-2. 재고현황 시트 컬럼

| 열 | 컬럼명 | 내용 |
|----|--------|------|
| A | PROD_CD | 품목코드 (이카운트 API) |
| B | BAL_QTY | 장부재고 (이카운트 API) |
| C | 조기출고수량 | 조기출고 시트에서 PROD_CD 매칭 합산 |
| D | 실제재고 | BAL_QTY - 조기출고수량 |
| E | 갱신시간 | API 호출 완료 시각 |

---

### 4-3. 조기출고 시트 컬럼

| 열 | 컬럼명 | 내용 |
|----|--------|------|
| A | PROD_CD | 품목코드 (담당자 입력) |
| B | 조기출고수량 | 출고 수량 (담당자 입력) |
| C | 입력시각 | 자동 기록 |

> **입력 규칙:** 조기 출고 발생 시만 입력. 동일 품목 복수 입력 시 합산 처리.

---

## 5. GAS 기능 목록

| 함수명 | 역할 |
|--------|------|
| `getEcountSession()` | 이카운트 Zone 조회 → 로그인 → SESSION_ID 반환 |
| `fetchInventory(sessionId)` | 재고현황 API 호출 → 품목별 BAL_QTY 배열 반환 |
| `getEarlyShipments()` | 조기출고 시트 데이터 읽기 → PROD_CD별 합산 Map 반환 |
| `updateInventorySheet()` | 재고현황 시트 전체 덮어쓰기 (메인 함수) |
| `resetEarlyShipments()` | 조기출고 시트 자정 초기화 |

## 6. 이카운트 API 스펙

### 6-1. Zone 조회

- **URL**: `https://oapi.ecount.com/OAPI/V2/Zone`
- **Method**: POST
- **파라미터**: `{ "COM_CODE": "[회사코드]" }`

### 6-2. 로그인

- **URL**: `https://oapi{ZONE}.ecount.com/OAPI/V2/OAPILogin`
- **Method**: POST
- **파라미터**: `COM_CODE`, `USER_ID`, `API_CERT_KEY`, `LAN_TYPE`, `ZONE`

### 6-3. 재고현황 조회

- **URL**: `/OAPI/V2/InventoryBalance/GetListInventoryBalanceStatus?SESSION_ID={SESSION_ID}`
- **Method**: POST
- **파라미터**:

| 파라미터 | 값 | 필수 |
|---------|-----|------|
| BASE_DATE | 호출일자 (YYYYMMDD) | Y |
| WH_CD | 200 | - |
| ZERO_FLAG | N | - |

- **응답 필드**: `PROD_CD`, `BAL_QTY`

> **주의:** WH_CD=200 지정해도 전체 창고 합산값 반환 확인됨. 인천창고가 유일 출고창고임을 전제로 사용.

| 트리거 | 함수 | 주기 |
|--------|------|------|
| 시간기반 | `updateInventorySheet` | 매 1시간 |
| 시간기반 | `resetEarlyShipments` | 매일 자정 (00:00) |

## 8. 개발 순서

1. Google Sheets 신규 생성 → 시트 2개 구성
2. GAS 신규 프로젝트 생성 → Sheets 연결
3. `getEcountSession()` 구현 및 테스트
4. `fetchInventory()` 구현 및 실데이터 확인
5. `getEarlyShipments()` 구현
6. `updateInventorySheet()` 통합 테스트
7. `resetEarlyShipments()` 구현 및 트리거 설정
8. 실데이터 검증 후 배포

---

## 9. 주의사항

- 이카운트 API 1일 허용량: **5,000건** (현재 사용량 확인 필요)
- 1시간 연속 오류 제한: **30건**
- SESSION_ID는 매 호출마다 갱신 권장 (세션 만료 방지)
- 조기출고 시트 입력 후 재고현황은 **다음 1시간 갱신 시** 반영됨
