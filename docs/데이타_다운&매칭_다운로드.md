> 파일: `scripts/p7_gas.js` (GAS 배포본: `gas-push/Code.js`)
> 최종 업데이트: 2026-03-24

---

## 1. 전체 흐름

```
[이카운트]                [카페24]
    │                        │
    ▼                        ▼
fetchEcountPrices()    buildCafe24Cache()
    │                        │
    ▼                        ▼
이카운트 가격 MAP       [카페24상품] 시트
(PROD_CD → 가격)       (custom_variant_code → product_no, variant_code, price)
    │                        │
    └──────────┬─────────────┘
               ▼
         syncPrices()
         [매핑테이블] 비교
         가격 변동 항목만 카페24 PUT
```

## 2. 카페24 상품 다운로드 — buildCafe24Cache()

### 실행 방법
GAS 편집기 → 함수 선택 `buildCafe24Cache` → 실행

### 동작 순서
1. ScriptProperties 리셋 (C24_CACHE_OFFSET=0, C24_CACHE_DONE=false)
2. [카페24상품] 시트 clearContents() 후 헤더 재작성
3. 카페24 Admin API 호출 — 판매중 상품 전체 순회 (100건씩 페이지)
4. 각 상품의 variants 조회 → custom_variant_code 있는 항목만 저장
5. 완료 시 저장/전체/미설정 건수 로그 출력

### API

#### 상품 목록 조회 (100건씩)
```
GET https://{mallId}.cafe24api.com/api/v2/admin/products
  ?selling=T
  &limit=100
  &offset={N}
  &fields=product_no,product_code,product_name
```

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| selling=T | T | 판매중 상품만 (전체 903건 → 약 663건) |
| limit | 100 | 페이지당 최대 건수 |
| offset | 0, 100, 200... | 페이지 오프셋 |

#### variants 조회 (상품당 1회)
```
GET https://{mallId}.cafe24api.com/api/v2/admin/products/{product_no}/variants
  ?fields=variant_code,custom_variant_code,additional_amount
```

### 필터링 조건
- custom_variant_code 비어있거나 null → 제외
- custom_variant_code 값이 있는 variant만 → 저장

### 저장 시트: [카페24상품]

| 컬럼 | 필드 | 설명 |
|------|------|------|
| A | product_no | 카페24 상품 번호 |
| B | product_code | 카페24 상품 코드 |
| C | product_name | 상품명 |
| D | custom_variant_code | 이카운트 PROD_CD와 매칭 키 |
| E | variant_code | 카페24 variant 코드 (가격 PUT 시 사용) |
| F | additional_amount | 카페24 현재 추가금액(가격) |

### 완료 로그 예시
```
[buildCafe24Cache] offset=0 variants=87 저장=52 (누적 저장 52건 / 전체 variants 87건)
[buildCafe24Cache] offset=100 variants=91 저장=65 (누적 저장 117건 / 전체 variants 178건)
...
[buildCafe24Cache] ✅ 완료 — 저장 480건 / 전체 variants 663건 / 미설정 제외 183건
```

## 3. 이카운트 가격 다운로드 — fetchEcountPrices()

### API
```
POST https://{zone_url}/OAPI/V2/InventoryBasic/GetBasicProductsList?SESSION_ID={sid}
```
로그인(ecLogin())으로 SESSION_ID 획득 후 호출.

### 필터링 조건
```javascript
if (cd.includes('(1)') && des.includes('◈')) { ... }
```
- PROD_CD에 `(1)` 포함 — 다산3833 제품 코드 규칙
- PROD_DES에 `◈` 포함 — 카탈로그 등록 대상 품목 표시

### 반환 데이터
```javascript
{
  prices: { "(1)200A자이01일반93": 12000, ... },         // PROD_CD → 가격
  descriptions: { "(1)200A자이01일반93": "자이로폼◈", ... } // PROD_CD → 품목명
}
```

### 가격 필드
[설정] 시트 ECOUNT_PRICE_FIELD 키 값 (기본: OUT_PRICE2)

## 4. 매칭 키

| 시스템 | 필드 | 예시 |
|--------|------|------|
| 이카운트 | PROD_CD | (1)200A자이01일반93 |
| 카페24 variants | custom_variant_code | (1)200A자이01일반93 |

두 값이 일치할 때 매핑 성공.
카페24 관리자 → 해당 상품 → 옵션 → 자체 상품코드(옵션) 필드에 이카운트 PROD_CD를 직접 입력해야 한다.

## 5. 매핑테이블 & 가격 동기화 — syncPrices()

### [매핑테이블] 시트 구조

| 컬럼 | 필드 | 설명 |
|------|------|------|
| A | custom_variant_code | 이카운트 PROD_CD |
| B | product_no | 카페24 상품번호 |
| C | variant_code | 카페24 variant 코드 |
| D | cached_price | 마지막 동기화 가격 (VAT 포함) |
| E | 최근업데이트 | 타임스탬프 |
| F | 결과 | 성공/스킵/실패/미매핑 |

### 동기화 흐름

```
이카운트 PROD_CD별 가격 순회
  │
  ├─ [매핑테이블]에 존재?
  │   ├─ YES → cached_price와 비교
  │   │          ├─ 변동 없음 → 스킵
  │   │          └─ 변동 있음 → 카페24 PUT
  │   │
  │   └─ NO  → [카페24상품] 캐시에서 조회
  │             ├─ 있음 → 자동등록 후 가격 비교
  │             └─ 없음 → 미매핑 기록
```

### 가격 계산
```
카페24 업데이트 가격 = Math.round(이카운트 가격 × 1.1)   // VAT 10% 포함
```

### 카페24 PUT API
```
PUT https://{mallId}.cafe24api.com/api/v2/admin/products/{product_no}/variants/{variant_code}
Body: { "request": { "variant": { "additional_amount": "12100" } } }
```
- variant_code 없는 단품: PUT /products/{product_no} 직접 호출

## 6. 토큰 관리

| 소스 | 위치 | 갱신 주체 |
|------|------|---------|
| access_token | 모니터링 시트 [설정] → p7_gas.js 오버레이 | monitoring-gas refreshCafe24Token() 1시간 트리거 |

## 7. 미매핑 발생 시 조치

**원인**: 카페24 variant의 자체 상품코드(옵션) 필드 미입력

**해결 방법**:
1. 카페24 관리자 → 해당 상품 → 옵션 → 자체 상품코드 란에 이카운트 PROD_CD 입력
2. buildCafe24Cache() 재실행
3. 다음 syncPrices() 실행 시 자동 매핑
