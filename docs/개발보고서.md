# 개발보고서

본 문서는 이카운트(ERP)와 카페24 쇼핑몰 간 가격 동기화 시스템의 설계, 구현, 운영 방법을 정리한 보고서입니다. 비개발자도 이해할 수 있도록 전체 흐름을 설명하고, 재개발 및 유지보수가 가능한 수준으로 작성했습니다.

## 1) 프로젝트 목표
- 이카운트(ERP)의 품목 가격(OUT_PRICE2)을 카페24 상품/옵션 가격에 자동 동기화
- 변경분만 빠르게 업데이트하여 실행 시간 단축 및 API 호출 최소화
- 운영자가 실수 없이 재실행 가능한 안정적인 자동화 환경 구축

## 2) 전체 시스템 아키텍처
- **데이터 소스**: 이카운트 ERP API
- **동기화 대상**: 카페24 Admin API (상품/옵션)
- **중간 매핑**: 품목코드(PROD_CD) ↔ 카페24 variant/product 매핑 테이블
- **자동화 실행**: Google Apps Script(GAS) + 시간 트리거
- **보조 도구**: 로컬 Python 스크립트(데이터 가공/검증/샘플 테스트)

흐름 개요:
1. 이카운트 API 로그인 → 가격 목록 획득
2. 매핑테이블 기반으로 변동 항목만 추출
3. 카페24 API로 옵션/상품 가격 업데이트
4. 실행 결과를 매핑테이블과 로그에 기록

## 3) Phase별 개발 내용 및 결과

### Phase 1: API 연동 및 기본 동작 검증
- 이카운트/카페24 인증 및 기본 호출 성공 여부 검증
- 샘플 데이터 다운로드 및 구조 확인
- API 토큰 갱신 로직 구축
- 결과: API 인증과 기본 GET/PUT 호출 가능

### Phase 2: 매핑 생성 및 데이터 정제
- 이카운트 품목 목록 정제 및 필터링
- 카페24 상품 목록과 매칭 로직 구축
- 매핑 테이블 생성(CSV/JSON)
- 결과: 이카운트 품목 ↔ 카페24 상품 간 기본 매핑 확보

### Phase 3: 가격 동기화 로직 구현
- 매핑테이블 기반 가격 비교 로직 구현
- 변동 항목만 타겟 업데이트
- 에러 로깅 및 실행 로그 누적
- 결과: 가격 업데이트 자동화 완료

### Phase 4: GAS 자동화 및 실행시간 최적화
- GAS 기반 동기화 로직 구현
- 전체 상품 스캔 제거, 변동 항목만 업데이트
- 재시도/시간 가드/429 대응 등 안정성 강화
- 결과: 실행시간 단축 및 운영 안정성 확보

## 4) 핵심 로직 설명

### 4.1 인증 (Authentication)
- 이카운트: SESSION_ID 기반 로그인
- 카페24: OAuth Access/Refresh 토큰 기반
- 토큰 만료 시 자동 갱신 처리

### 4.2 수집 (Collection)
- 이카운트: `GetBasicProductsList`로 가격/품목코드 수집
- 카페24: 매핑된 product/variant만 조회

### 4.3 매핑 (Mapping)
- 기본 매핑: `custom_variant_code` == `ecount_prod_cd`
- 폴백 매핑: 구형 매핑테이블 CSV(`data/mapping_table.csv`) 기반 `product_no` 참조
- 매핑 실패 시 상품 가격(P6 방식) 직접 업데이트로 폴백

### 4.4 업데이트 (Update)
- 표준: `PUT /products/{product_no}/variants/{variant_code}`
- 단품/매핑 실패: `PUT /products/{product_no}` 가격 직접 갱신
- 실제 동기화는 **가격 변동분만** 처리

### 4.5 자동화 (Automation)
- GAS `syncPrices` 함수가 주기적으로 실행됨
- 결과는 매핑테이블 및 실행로그 시트에 기록

## 5) 문제 및 해결 과정

1. **API 호출량 초과(429)**
   - 원인: 전체 상품 스캔 시 과도한 호출 발생
   - 해결: 전체 스캔 제거, 변동 항목만 타겟 업데이트

2. **variant_code 매칭 실패**
   - 원인: custom_variant_code 누락 또는 구형 데이터
   - 해결: 구형 매핑테이블 CSV 폴백 + variant 조회 후 자동 등록

3. **GAS 실행 6분 제한 초과**
   - 원인: 전체 스캔 및 불필요한 반복 호출
   - 해결: 실행 시간 가드, 지연 시간 최적화

4. **Address unavailable 오류**
   - 해결: `_rawPut`에서 5초 후 1회 재시도 후 스킵

## 6) 주요 용어 설명
- **PROD_CD**: 이카운트 품목 코드
- **OUT_PRICE2**: 이카운트 단가(가격)
- **product_no**: 카페24 상품 고유 번호
- **variant_code**: 카페24 옵션(variant) 코드
- **custom_variant_code**: 카페24 옵션에 저장된 커스텀 코드(이카운트 PROD_CD와 매칭용)
- **매핑테이블**: 이카운트 품목 ↔ 카페24 상품/옵션 관계를 저장한 표

## 7) 파일 구조

- 루트: API/유틸 스크립트, 실행 진입점
- `scripts/`: 단계별 실행 스크립트 및 업데이트 도구
- `data/`: 원본/중간/결과 데이터
- `config/`: 설정 및 시크릿
- `docs/`: 문서
- `gas-push/`: Apps Script 배포용 코드

자세한 파일 역할은 `docs/필요한파일.md`를 참고하세요.

## 8) 운영 방법

1. **카페24 토큰 유효성 확인**
   - 토큰 만료 시 갱신 필요

2. **매핑테이블 업데이트**
   - 초기에는 구형 매핑 데이터 활용
   - 신규 품목은 자동 등록 또는 보조 매핑 추가

3. **GAS 동기화 실행**
   - `syncPrices` 실행 (수동 or 트리거)

4. **결과 확인**
   - 매핑테이블 시트 F열 결과 확인
   - 실행로그 시트에서 실패/오류 확인
