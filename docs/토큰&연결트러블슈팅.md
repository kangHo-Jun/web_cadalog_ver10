# Cafe24 토큰/연결 오류 해결 문서 (통합)

이 문서는 Cafe24 API 토큰/연결 문제와 자동화 복구 흐름을 한 곳에 정리한 **즉시 해결용 문서**입니다.
문제가 발생했을 때 이 문서만 참고하면 원인 파악 → 조치 → 정상 확인까지 진행할 수 있습니다.

---

## 0) 빠른 진단 순서 (3분 루틴)
1. **Vercel 환경변수 확인**: `KV_REDIS_URL`, `CAFE24_CLIENT_ID`, `CAFE24_CLIENT_SECRET`, `CAFE24_ACCESS_TOKEN`, `CAFE24_REFRESH_TOKEN`, `MALL_ID`
2. **Redis 연결 확인**: `/api/debug-snapshot` 호출 → `status: OK` 확인
3. **API 호출 확인**: `/api/products` 또는 `/api/sync-products` 호출 → HTTP 200 확인

---

## 1) 시스템 구조 이해
[구글시트 GAS] → [Vercel API] → [Redis] → [Cafe24 API]
↑
토큰 저장/조회

- 토큰은 Redis에 저장되며, Vercel 서버가 Redis에서 토큰을 읽어 Cafe24 API 호출
- Redis 연결 실패 = 토큰 조회 불가 = 401 오류
- 토큰 만료(2시간) 시 refresh_token으로 자동 갱신 시도

## 2) 주요 증상 & 즉시 해결

### A. invalid_client_secret (401)
오류 예시:
`{"error":"invalid_client","error_description":"Invalid client_secret"}`

**원인**
- Redis에 토큰이 없어서 refresh 시도 → client_secret 불일치
- Cafe24 파트너센터에서 Client Secret 재발급 후 Vercel 미반영

**해결**
1. `python3 scripts/reauth_cafe24.py` 실행
2. Redis에 새 토큰 직접 저장 (아래 명령 참고)
3. Vercel 환경변수 업데이트 후 Redeploy

---

### B. Redis 연결 오류
**원인**
- Vercel 환경변수에 `KV_REDIS_URL` 미설정 또는 잘못된 값
- 코드에서 읽는 변수명: `KV_REDIS_URL` (REDIS_URL 아님)

**해결**
Vercel → Settings → Environment Variables →
`KV_REDIS_URL` 값 확인/추가

---

### C. 스냅샷 상품수 0
**원인**
- Redis에 스냅샷 데이터 없음
- sync-products API 미실행

**해결**
```
curl https://web-cadalog-ver10.vercel.app/api/sync-products
```

---

### D. API 버전 오류
오류 예시:
`2024-03-01 version you requested is not available`

**해결**
`X-Cafe24-Api-Version: 2025-12-01` 고정

---

### E. DNS/호스트 문제
증상: `Could not resolve host`

**해결**
```
sudo sed -i '' '/daesan3833.cafe24api.com/d' /etc/hosts
sudo dscacheutil -flushcache
nslookup daesan3833.cafe24api.com 8.8.8.8
```
필요 시 확인된 IP로 재등록:
```
sudo sh -c 'echo "IP_ADDR daesan3833.cafe24api.com" >> /etc/hosts'
sudo dscacheutil -flushcache
```

---

### F. 네트워크/방화벽 차단
증상: HTTPS 연결 즉시 실패 (`Operation not permitted` 등)

**확인**
```
curl -I -m 10 https://daesan3833.cafe24api.com | head -n 5
```

## 3) 토큰 재발급 절차 (확정 루틴)

### Step 1: 토큰 재발급
```bash
python3 scripts/reauth_cafe24.py
```

### Step 2: Redis에 토큰 저장
```bash
node -e "
const { createClient } = require('redis');
const token = require('./data/token_state.json');
const client = createClient({ url: process.env.KV_REDIS_URL });
client.connect().then(async () => {
  await client.set('cafe24_tokens', JSON.stringify({
    access_token: token.access_token,
    refresh_token: token.refresh_token,
    expires_at: Date.now() + 2*60*60*1000,
    refresh_token_expires_at: Date.now() + 14*24*60*60*1000
  }));
  console.log('✅ Redis 저장 완료');
  await client.disconnect();
});
"
```

### Step 3: Vercel 환경변수 업데이트
```
CAFE24_ACCESS_TOKEN = (새 값)
CAFE24_REFRESH_TOKEN = (새 값)
```

### Step 4: Vercel 재배포
```bash
npx vercel --prod
```

## 4) 자동화 상태 (구현 완료 요약)

### ✅ access_token 자동 갱신
- 위치: `src/app/api/cron/check-token-expiry/route.ts`
- 만료 30분 전 갱신 + 실패 시 이메일 알림
- 검증 응답 예: `{"status":"ok","access_token_expires_in_min":119}`

### ✅ refresh_token 하루 전 알림
- `ALERT_DAYS = 1` 적용
- GAS 알림 조건도 1일 전 기준으로 변경

### ✅ Redis 유실 자동 복구
- `saveTokens()`에서 Redis 저장 + Vercel env 업데이트
- `getTokens()` env fallback 시 Redis 자동 재저장

## 9. 자동 갱신 실패 대응 (2026-03-16 추가)

### 문제
- CAFE24_CLIENT_SECRET 값이 잘못되면 자동 갱신 실패
- 실패 후 복구 로직 없어 서비스 중단

### 해결: 재시도 로직 추가
- 파일: src/app/api/cron/check-token-expiry/route.ts
- 갱신 실패 시 15분 간격으로 최대 3회 자동 재시도
- 3회 모두 실패 시 이메일 발송

### 재시도 흐름
갱신 실패 (1회)
→ 15분 후 재시도 (2회)
→ 15분 후 재시도 (3회)
→ 3회 모두 실패 시 이메일 발송

### 근본 원인
- CAFE24_CLIENT_SECRET이 Vercel 환경변수에 잘못 설정된 경우
- Cafe24 파트너센터에서 Secret 재발급 후 Vercel 미반영
- 이 경우 사람이 직접 파트너센터 확인 후 재설정 필요

### 예방
- Vercel 환경변수 CAFE24_CLIENT_SECRET 주기적 확인
- 자동 갱신 실패 이메일 수신 시 즉시 파트너센터 확인
