## 1. 웹카다로그 품목·가격 출력 메카니즘 (중요)

### 1) 품목 데이터 생성 경로
1. `/api/sync-products`가 Cafe24 Admin API에서 상품을 조회한다.
2. `embed=options,variants`로 옵션·변형 정보를 함께 받는다.
3. 상품을 부모/자식(옵션) 구조로 변환해 `GroupedProduct`를 만든다.
4. 결과를 Redis `catalog:snapshot:v1`에 저장한다.
5. 프론트는 `/api/debug-snapshot`에서 Redis 스냅샷을 읽어 렌더링한다.

즉, **웹카다로그는 Cafe24 API를 직접 호출하지 않는다.**
**Redis 스냅샷이 단일 진실 소스**다.

### 2) 가격 데이터 생성 경로
1. 구글 시트를 gviz API로 조회한다.
2. `custom_variant_code → 가격` 형태의 `priceMap`을 만든다.
3. `/api/sync-products`에서 `priceMap`을 로드해 각 자식 품목에 가격을 넣는다.

### 3) 실제 매칭 키
- Cafe24 변형(variant) 데이터의 `custom_variant_code`
- 구글 시트의 `custom_variant_code`
- priceMap의 키

즉, **`custom_variant_code`가 가격 매칭의 연결고리**다.

### 4) 프론트에서 가격 표시 방식
- ChildItem.price는 **부가세 포함가** 기준으로 저장됨
- 프론트는 `formatSupplyPrice`로 **부가세 제외 금액**을 표시
- 가격이 없으면 “가격문의” 표시

## 2. 웹카다로그 품목·가격 모니터링 (시트 연동 아이디어)

목표:
- 구글 시트의 `custom_variant_code`(E열)을 기준으로
- 웹카다로그에 실제 출력되는 품목명과 가격을 시트에 기록

매칭 기준:
- 연결고리: `custom_variant_code`
- Cafe24 variants의 `custom_variant_code` ↔ 시트 E열

데이터 소스:
- 웹카다로그 출력 기준은 `/api/debug-snapshot` (Redis 스냅샷)
- `parentName`, `children[].name`, `children[].price` 활용

시트 매핑 예시:
- G열: `children[].name` (웹카다로그 표시 품목명)
- H열: `children[].price` (웹카다로그 표시 가격 값)

주의:
- `children[].price`는 부가세 포함가 기준으로 저장됨
- 시트에 부가세 제외 금액을 넣고 싶다면 `/1.1` 변환 필요

## 3. 전체 데이터 흐름

구글 시트 (카페24 실시간 수집)
    ↓ custom_variant_code + additional_amount
/api/prices (price-map.ts)
    ↓ priceMap {custom_variant_code: price}
/api/sync-products (Redis 저장)
    ↓ GroupedProduct { children[].price }
Redis (catalog:snapshot:v1)
    ↓
/api/debug-snapshot
    ↓
프론트엔드 Phase1CatalogView → 상품 카드 가격 표시

## 4. 부모-자식 로직 (2026-02-19 기준)

참조: `docs/로직/부모자식_로직수정_ver0219.md`

핵심 변경 사항:
- `has_option` 경로는 `product.options?.has_option` 사용
- 카테고리 번호는 `QUOTE_CATEGORY_NOS` 사용
- 상품 조회 시 `display: 'T'`로 진열 상품만 포함
- 상품명은 HTML 태그만 제거하고 절삭하지 않음

자식 구성 규칙:
- Case A (옵션 상품, `has_option: 'T'`)
  - 부모: `product_code`
  - 자식명: `options.options[0].option_value`의 `value` 또는 `option_text`
  - 자식가격: `product.price + variant.additional_amount`
  - 자식코드: `variant_code`
- Case B (단일 상품, `has_option: 'F'`)
  - 부모 자체를 자식 1개로 반환
  - 가격: `product.price`
  - `isSingle: true`
